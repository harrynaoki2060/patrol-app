# GAS デプロイ手順

## Phase 1 完了条件（4つ全て必須）

- [ ] ① `setupApiKey()` 実行成功
- [ ] ② `testHealth()` がエディタのログに `{"ok":true,...}` を返す
- [ ] ③ Web App デプロイ後、ブラウザで health URL を開いて `{"ok":true,...}` が返る
- [ ] ④ アプリの設定画面に URL と API キーを入力して「接続テスト」が成功する

---

## 重要: スクリプトの作成方法

**このスクリプトは必ず Google スプレッドシートの中から作成してください。**
script.google.com で単独作成（スタンドアロン）にするとスプレッドシートに接続できずエラーになります。

---

## ステップ 1: スプレッドシートを作成

1. https://sheets.google.com を開く
2. 「空白のスプレッドシート」を新規作成
3. タイトルを「安全パトロール」に変更

### Google Spreadsheet ID の取得方法

スプレッドシートを開いたときのブラウザ URL に含まれています:

```
https://docs.google.com/spreadsheets/d/【ここがSpreadsheet ID】/edit#gid=0
```

例: `1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms`

> Phase 2 以降でアプリの設定画面に入力するために控えておいてください。
> スクリプト自体には入力不要です（コンテナバインド方式のため自動連携）。

---

## ステップ 2: Apps Script を開く

1. スプレッドシートのメニュー →「**拡張機能**」→「**Apps Script**」をクリック
2. 新しいタブでエディタが開く
3. プロジェクト名を「安全パトロールAPI」に変更（左上の「無題のプロジェクト」をクリック）

---

## ステップ 3: コードを貼り付ける

1. エディタ左ペインの `コード.gs` をクリック
2. 既存の内容（`function myFunction() {}`）を**全て削除**
3. [`gas/Code.gs`](Code.gs) の内容を**全てコピー&ペースト**
4. **Ctrl+S で保存**

> 貼り付け後、先頭行が以下になっていることを確認してください:
> ```
> // Google Apps Script Web App
> ```

---

## ステップ 4: API キーを登録する（手動実行）

1. エディタ上部の関数プルダウンで **`setupApiKey`** を選択
2. コード内の `patrol-2026-change-me` を**自分のキーに書き換え**
   - 例: `patrol-nishino-2026`
   - 英数字・ハイフンのみ推奨、スペースなし
3. 「**実行**」ボタン（▶）をクリック
4. 初回は「承認が必要です」ダイアログ → 「**権限を確認**」→ 自分のアカウントを選択 → 「**許可**」
5. 実行ログに `SUCCESS: API_KEY saved as: patrol-nishino-2026` と出れば成功

> **エラーが出た場合**: `patrol-2026-change-me` のままにしていないか確認してください。
> コード内のキーを変更してから再実行してください。

---

## ステップ 5: 動作確認（testHealth 実行）

1. 関数プルダウンで **`testHealth`** を選択
2. 「実行」をクリック
3. 下部の「実行ログ」に以下が表示されれば ✅ 成功:

```json
{"ok":true,"spreadsheetId":"1Bxi...","spreadsheetName":"安全パトロール","timestamp":"2026-07-23 14:30:00"}
```

> エラーが出た場合:
> - `Spreadsheet not found` → スクリプトをスプレッドシート内から作成していない
>   → スプレッドシートのメニュー「拡張機能」から作り直してください

---

## ステップ 6: Web App としてデプロイする

1. エディタ右上「**デプロイ**」→「**新しいデプロイ**」をクリック
2. 左上の歯車アイコン（⚙）→「**ウェブアプリ**」を選択
3. 以下の通りに設定する:

| 項目 | 設定値 |
|---|---|
| 説明 | `安全パトロール v1` |
| 次のユーザーとして実行 | **自分（メールアドレス）** |
| アクセスできるユーザー | **全員** |

4. 「**デプロイ**」をクリック
5. 表示された **ウェブアプリ URL** をコピーして手元に保存

```
https://script.google.com/macros/s/AKfycbXXXXXXXXXX/exec
```

---

## ステップ 7: ブラウザで最終確認

以下の URL をブラウザのアドレスバーに入力して開く
（`【デプロイID】` と `【APIキー】` は自分のものに置き換え）:

```
https://script.google.com/macros/s/【デプロイID】/exec?action=health&key=【APIキー】
```

以下のレスポンスが返れば **Phase 1 完了**:

```json
{"ok":true,"spreadsheetId":"...","spreadsheetName":"安全パトロール","timestamp":"2026-07-23 ..."}
```

> レスポンスがない / HTML が返る場合:
> - デプロイ設定の「アクセスできるユーザー」が「全員」になっているか確認
> - URL の末尾が `/exec` になっているか確認（`/dev` ではない）

---

## コードを変更した後の再デプロイ（必須）

**コードを変更した場合は必ず新しいバージョンをデプロイしてください。**
コードを保存しただけでは、既存のデプロイには反映されません。

1. エディタ右上「**デプロイ**」→「**デプロイを管理**」をクリック
2. 既存のデプロイ行の右端「**鉛筆アイコン**（編集）」をクリック
3. 「バージョン」プルダウンで「**新バージョン**」を選択
4. 「**デプロイ**」をクリック

> URL は変わりません。同じ URL に新しいコードが反映されます。

---

## トラブルシューティング

| 症状 | 原因と対処 |
|---|---|
| `{"error":"Forbidden","code":403}` | `?key=` が間違っている。API キーを確認 |
| `{"error":"Spreadsheet not found","code":500}` | スタンドアロンで作成している。スプレッドシートの拡張機能から作り直す |
| `{"error":"Unknown action","code":400}` | `?action=health` のスペルを確認 |
| HTML（ログイン画面）が返る | デプロイの「アクセス」が「全員」になっているか確認 |
| `{"error":"You must change the key value","code":500}` | `setupApiKey()` でキーを変更せずに実行した |
| 古いコードのまま動いている | 再デプロイで「新バージョン」を選択したか確認 |
